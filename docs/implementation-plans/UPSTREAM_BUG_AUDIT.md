# Upstream ScanCode Bug Audit

Status: 🟡 Active cross-cutting audit reference

This document keeps a Provenant-side watchlist for upstream ScanCode bugs and open PRs that still matter to us. It is intentionally action-first: the main table below is ordered from highest to lowest priority, folds related upstream issues and PRs into the same row, and only keeps work that still looks unresolved or still merits targeted investigation on the Rust side.

The trailing table is the review residue: upstream bugs and PRs that were checked, but either do not apply to Provenant, are already covered locally, or are being intentionally held for compatibility reasons.

## How to Work an Item

For both humans and agents, the expected workflow is:

1. Read the upstream issue and any relevant upstream PRs before changing code.
2. Study the Provenant side deeply first: related local issues, current code paths, existing tests, and any nearby plan/improvement docs.
3. Reproduce the problem locally where practical, then turn that reproducer into the smallest durable regression coverage that fits the change: unit, parser/integration, scanner-level, or golden tests.
4. Implement the fix or feature end-to-end rather than stopping at partial plumbing.
5. Run the narrowest relevant verification for the changed surface, including golden tests when output parity or fixture-backed behavior is the point of the change.
6. If Rust intentionally diverges from current upstream behavior, document that decision clearly instead of leaving it implicit.
